# Scripts

Place repeatable project commands here, such as environment setup, build, validation, test-data generation, migrations, documentation checks, and release verification.

Scripts should:

- Be safe to run from a documented working directory
- Fail clearly and return a non-success status when validation fails
- Avoid printing secrets or protected data
- Support non-interactive execution when practical
- State prerequisites and side effects
- Be used by both local contributors and automated validation where possible

Document the main commands in `AGENTS.md`.
