## Changelog policy

Usually include:

- New user-facing behavior
- Changed workflows or externally visible interfaces
- Defect fixes users or operators would notice
- Deprecations, removals, and breaking changes
- Operational or security changes that should be communicated

Usually exclude:

- Formatting-only changes
- Internal refactoring with no observable effect
- Test-only changes
- Routine documentation corrections
- Individual implementation tasks that do not create a releasable outcome
