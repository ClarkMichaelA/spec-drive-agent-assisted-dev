## How to use this file

A roadmap organizes delivery around outcomes and evidence. It is not a detailed task list. Dates should be commitments only when the team has enough information to make them responsibly.

## Roadmap principles

- Deliver a thin, end-to-end walking skeleton early.
- Reduce the highest uncertainty and risk before scaling implementation.
- Prefer demonstrable outcomes over layers of incomplete infrastructure.
- Do not pull later scope into the current milestone without an explicit change decision.
