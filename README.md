This scaffold and playbook provide an AI vendor-neutral framework for spec-driven, agent-assisted software development, giving teams a reusable repository structure, documentation templates, review gates, and execution patterns.  Together, they help turn a high-level project idea into user journeys, requirements, architecture, decisions, roadmaps, tasks, implementation loops, testing evidence, and delivery handoffs while keeping human review and traceability at the center.
# How to Use
Start with the scaffold’s `README.md`, then keep the playbook open alongside it while completing `docs/PROJECT.md`.
# Reusable repository scaffold
The spec-driven-development-playbook contains 35 starter files, including:
- A vendor-neutral `AGENTS.md` working agreement
- `PROJECT.md` 
- `user_journeys/` (index + one file per journey)
- `REQUIREMENTS.md`
- `ARCHITECTURE.md`    
- `ROADMAP.md`    
- `TASKS.md`    
- `HANDOFF.md`    
- `CHANGELOG.md`- 
- Assumption and risk registers    
- A lightweight traceability map    
- Security, operations, testing, data-model, and API templates    
- Architecture Decision Record templates    
- Active and completed implementation-plan folders    
- Structured review templates    
- Placeholders for source code, tests, scripts, and automated quality gates    

The templates explain what belongs in each section, what does not belong there, how the artifacts relate to each other, and what must be reviewed before proceeding.
# Plain-language development playbook
The spec-driven-repository-scaffold contains 19 files covering the full lifecycle:
1. Lifecycle overview    
2. Project initiation    
3. User journeys and discovery    
4. Requirements development    
5. Decisions and architecture    
6. Roadmaps, implementation plans, and tasks    
7. The bounded implementation loop    
8. Testing, independent review, and release    
9. Project memory and handoffs    
10. Change control and maintenance    
11. Common failure modes    
12. Scaling from a solo project to parallel work    

It also includes:
- A copy-ready prompt library with 28 prompts    
- Stage-gate and review checklists    
- A recommended sequence of AI-assisted working sessions    
- A fictional, explicitly labeled example showing traceability from project outcome through requirement, architecture, task, and test    
- A plain-language glossary    
- Reusable end-of-session and project-resumption prompts
